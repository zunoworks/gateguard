# Claude Codeの出力品質を2割上げる、たった2コマンドの仕掛け

## AIは「考えてから書く」と上手くなる

Claude Codeを使っていて、こんな経験はないだろうか。

- コードは動く。テストも通る。でも設計が浅い
- 既存のパターンを無視して、独自の書き方をしてくる
- データの日付フォーマットを確認せず、勝手にISO-8601だと仮定する
- 「大丈夫ですか？」と聞くと「問題ありません」と言う。問題があるのに

最後の項目が核心だ。**LLMに自己評価させても行動は変わらない。** 「ポリシーに違反していませんか？」と聞けば、常に「していません」と答える。これは我々が実際に検証して確認した事実だ。

ではどうするか。

**「評価しろ」ではなく「調べろ」と言えばいい。**

## GateGuard — 事実調査を強制するゲート

[GateGuard](https://github.com/zunoworks/gateguard)は、Claude Codeの`PreToolUse`フックを使って、AIがファイルを編集する前に「事実調査」を強制するツールだ。

仕組みは単純：

1. AIがファイルを編集しようとする → **ゲートが止める**
2. 「このファイルをimportしている全ファイルを列挙しろ」「実データを1件確認しろ」「ユーザーの指示を原文で引用しろ」と要求する
3. AIが事実を調べて提示する → **2回目の試行を許可する**

ポイントは、2回目の試行時にはAIの文脈に「調べた事実」が入っていること。これだけで出力が変わる。

## 証拠：A/Bテスト結果

同じタスクを、ゲートありとゲートなしの2つのAIエージェントに同時にやらせた。2つの独立したタスクで検証。

| タスク | ゲートあり | ゲートなし | 差 |
| --- | --- | --- | --- |
| 既存コードベースとの統合モジュール | **8.0 / 10** | 6.5 / 10 | +1.5 |
| Webhookバリデータ（データパース） | **10.0 / 10** | 7.0 / 10 | +3.0 |
| **平均** | **9.0** | **6.75** | **+2.25** |

採点基準は5項目（コード構造、エッジケース処理、既存パターン準拠、テスト品質、設計判断）× 各2点。

両方ともテストは通る。コードは動く。**差が出るのは設計の深さだ。** ゲートなしのAIは推測で書く。ゲートありのAIは調べてから書く。

これはテストでは検出できないバグだ。コードは動くが、設計が浅い。ファイルが10個のプロジェクトなら、この2点の差が積み重なって大きな手戻りになる。

## なぜ誰もやっていなかったのか

AI guardrails（ガードレール）の分野は活発だ。しかしほぼ全てが「出力を事後チェックする」方向に進んでいる。有害発言フィルタ、コードレビュー、リンター。出力が出てから直す。

hookで「止める」ことは誰でも思いつく。**しかし「止めた後に何をさせるか」まで設計した人がいなかった。**

- deny（止める）だけでは品質は変わらない
- deny + 自己評価（「大丈夫？」）も効かない（常に「大丈夫」と答える）
- **deny + 事実調査強制 + 証拠要求** — この3段構造が品質を変える

「自己評価は効かないが事実調査は効く」という発見と、その実装を同時にパッケージにしたのがGateGuardだ。

## 導入：2コマンド

```bash
pip install gateguard-ai
gateguard init
```

これだけ。Claude Codeを再起動すればゲートが有効になる。

設定ファイル`.gateguard.yml`が生成されるので、ゲートの有効/無効やカスタムメッセージは自由に変更できる。

## 何が起きるか

導入後、Claude Codeでファイルを編集しようとすると：

```
[GateGuard — 事実提示ゲート]

ファイル src/api/handler.py を変更する前に、以下の事実を調べて提示しろ:

1. このファイルをimport/requireしている全ファイルを列挙しろ（Grepで検索）
2. 変更が影響する公開関数・クラスを列挙しろ
3. このファイルが読み書きするデータファイルがあれば、実データ1件を確認し、
   実際のフィールド名・構造・日付フォーマットを提示しろ
4. ユーザーの指示を原文のまま引用しろ（指示の範囲を確認）

事実を提示してから、同じ操作をやり直せ。
```

AIはこの要求に従って`Grep`や`Read`を実行し、事実を集める。その後の2回目の編集は通る。しかしこの時点でAIは、1回目にはなかった文脈を持っている。

**調査の行為そのものが、自己評価では生まれなかった認識を作る。**

## チームで広げる方法

プロジェクトの`CLAUDE.md`に1行加えるだけで、そのリポジトリでClaude Codeを使う全員に自動で伝わる：

```markdown
## コード品質ゲート
このプロジェクトはGateGuardを使用しています。作業開始前に `pip install gateguard-ai && gateguard init` を実行してください。
```

宣伝不要。仕組みが勝手に広がる。
